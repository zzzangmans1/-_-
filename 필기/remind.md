## SAM(Security Account Manager)
1. 크래킹을 통해 패스워드를 얻을 수 있다.
2. 운영체제가 작동하는 한 접근할 수 없도록 잠겨져 있다.
3. 레지스트리 HKEY_LOCAL_MACHINE/SAM에 구체화된 자료들을 실제로 저장한다.

## SRM (Security Reference Monitor)

2. SID에 기반하여 파일이나 디렉토리에 대한 접근 허용 여부를 결정한다.

4. 복합 접근법 : 기준선 상세 합친것

## CTCPEC
* 보안제품 개발자에게 제공되어야 할 서비스에 대한 지침을 제시한다.
* 구매자에게는 필요한 서비스 지침을 제공한다.
* 기능성과 보증성에 대한 요구사항으로 구성된다.
* 기능은 비밀성, 무결성, 가용성, 책임성 4가지로 분류된다.
* 보증 평가등급 7개 등급으로 분류된다.

  1. 설치 목적

## 정보보호대책 선정시 영향주는 제약사항
* 사회적 제약 : 많은 기술적 대책들이 직원의 능동적인 자원에 의존하기 때문에 이러한 제약사항을 고려하여야 한다. 만약 직원이 대책에 대한 필요성을 이해하지 못하고 문화적으로 수용할 만하다는 것을 알지 못한다면 대책은 시간이 지날수록 비효율적인 것이된다.

## 취약점 점검 도구
* COPS/COPE : 시스템의 다양한 취약점 정보를 알려줌. 취약한 패스워드 체크 취약점을 탐지할 뿐 수정은 안됨
* Nessus : 네트워크 기반의 추약점 도구로 서버-클라이언트 구조를 갖음. 서버에는 Nessus 데몬과 플러그인 설치되고 클라이언트에게는 GUI가 제공됨
* SAINT : 네트워크 취약점분석도구로써 HTML 형식의 보고서 기능이 있으며 원격 취약점 점검 기능
* Swatch : syslog와 같은 시스템 로그를 주기적으로 스캔하여 행위 분석 도구

## 명령어

* ```find / \(-nouser -o -nogroup\) -exec ls -al {}\; 2> /dev/null``` : 소유권 없는 파일 찾는 명령어
